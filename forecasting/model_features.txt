=== NOTE: WHY BOTH SIN AND COS FOR CYCLICAL FEATURES ===

Using both hour_of_day_sin and hour_of_day_cos encodes the hour as a cyclical feature.

Why both are needed:
- Hour is cyclical: 23 and 0 are adjacent, but a raw number (0-23) makes them appear far apart.
- Sin and cos together uniquely represent each hour on a circle.
- Using only one would map multiple hours to the same value.

How it works:
- hour_of_day_sin = sin(2π × hour / 24)
- hour_of_day_cos = cos(2π × hour / 24)

This preserves the cyclical relationship (e.g., 23 and 0 are close) and gives the model a continuous, periodic representation of time.

The same logic applies to:
- weekday_sin and weekday_cos for the 7-day cycle (weekday = sin(2π × day_of_week / 7))
- month_sin and month_cos for the 12-month cycle (month = sin(2π × month / 12))

The weekend binary indicator (0=weekday, 1=weekend) captures weekday vs weekend patterns that may differ from the cyclical weekday encoding.
